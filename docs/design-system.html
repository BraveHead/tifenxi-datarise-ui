01 · Principles

四条设计原则

所有设计决策的判断依据。当规范不明确时，回到原则。

1一致性 Consistency

同类信息使用同一种承载方式。模块结构、颜色语义、间距节奏、交互反馈在全局保持同形同名。偏差必须有理由。

2克制 Restraint

能用描边不用阴影，能用中性不用彩色，能少一层不多一层。颜色和强调是稀缺资源，只给真正关键的信息。

3可读性 Legibility

数字使用等宽数字字体；正文最小 13px；图表必配图例；Tag 有边界；颜色永远不是唯一承载信息的通道（配合文字或图形）。

4层次 Hierarchy

通过字号、字重、颜色和位置建立信息主次。每个模块应能用一眼识别出：标题 → 上下文 → 结论 → 支撑数据。

06 · Module Rules

模块的结构约束

任何业务模块，无论数据类型，都应能映射到这套槽位。

模块槽位定义
外框
white + border-subtle + radius-lg + padding 20/24
① Header
必填：标题（fs-16/600）；选填：严重度 / 时间范围 Tag（右）
② Context Bar
选填；info-bg 浅底 + ⓘ 图标 + KV 列，最多 5 组
③ Section
≥ 1 个；编号 ① ② 置首；section-heading 下划线
④ View Switcher
选填；Tabs / Segment；切同数据多视图
⑤ Data Block
必填；Bar Row / Seg Bar / Table / Chart 之一
⑥ Legend
若含颜色编码必填；置小节标题右侧
⑦ Footer
选填；更新时间 · "查看全部 →" 链接
✓ DO
同类模块复用同一结构。标题左、Tag 右；Callout 只放元信息不放结论；编号 Section 从 ① 开始。图例紧贴所在 Section 的右上角。
✗ DON'T
不要把结论放进 Callout。不同模块使用不同的圆角、阴影、描边色。模块内用多种字号排版标题（只能 fs-16 + 600）。用纯色背景替代描边卡片。
ⓘ 灵活性边界
当业务模块只有单一数据块（如纯 KPI 区），可以省略 ②③④⑥，但 ① Header 和 ⑤ Data Block 必须保留。

① 名称 NAME
13px neutral-600；左上单行；可跟 ⓘ 提示。不加粗，与数值形成对比。
② 图标徽章 ICON
28×28 圆角 8px。淡彩底 + 深色 stroke；一张卡只能一个；按指标类别配色（情感/趋势/人次/警示）。
③ 数值 + 单位
30px semibold 等宽字；单位 13px neutral-500 跟随其后半字号。行对齐 baseline。
④ 同环比
11px 透明底。语义色由"变化方向 × 指标性质"决定：越高越好 ↑=绿；越低越好 ↑=红。
⑤ 趋势线
全宽贴底 44px；统一 brand-500；1.5px 描边；无网格无轴。体现节奏而非精度。
✓ DO
一行 4–5 张、高度与槽位对齐；风险指标用 .kpi-card.danger（左侧 3px 红竖条 + #fff1ea→#fff6f2 渐变底）并可挂 .infer-chip AI 归因入口；sparkline 全部贴底通栏。
✗ DON'T
不要用填色 chip 表示同环比（会与 danger 卡色冲突）；不要让趋势线出现在卡片中部留白处；不要混用不同字重/字号数值；不要用多彩趋势线——所有 sparkline 统一 brand 蓝。
