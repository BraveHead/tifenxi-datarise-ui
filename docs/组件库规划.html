01 · 公共基础组件库
@datarise/ui shadcn 基础 + 扩展
这一层完全不感知业务域，可跨产品（数启、Make 等）共享。现有 /src/components/ui/ 中的 47 个 shadcn 组件构成基础，另需新建 9 个高频复用的扩展组件。所有组件必须同时支持 Light / Dark。
Dark Mode 实现方式：globals.css 已定义完整的 .dark {} token 覆盖（background / card / border / primary 等）。公共组件只需使用 CSS 变量（var(--background)、var(--border)），不应硬编码任何颜色值，即可自动适配。需重点审计的是图表颜色（--chart-1~5 在 dark 下已调亮）和阴影（--elevation-sm dark 下加深至 0.3 透明度）。

03 · Dark Mode 适配要点
主题切换机制 & 审计重点
globals.css 已定义完整双主题 token，切换机制为在 <html> 上切换 .dark class（next-themes 管理）。以下是需要特别关注的适配差异。
Light Mode tokens
--background#FFFFFF
--card#FFFFFF
--primary#1F61E8
--border#E4E5E8
--muted-foregroundrgba(2,8,19,.7)
--chart-1#1F61E8
--elevation-smrgba(0,0,0, .10)
Dark Mode tokens
--background#121212
--card#262626
--primary#4F8CFF ↑调亮
--border#404040
--muted-foregroundrgba(255,255,255,.6)
--chart-1#4F8CFF ↑调亮
--elevation-smrgba(0,0,0, .30) ↑加深
适配场景 风险等级 说明 & 建议
TopNavBar 硬编码色 高风险 TopNavBar 当前写死 bg-slate-900，dark 模式下与 --background: #121212 不一致。应改为 bg-[var(--sidebar)] 或 bg-background。
热力图矩阵色 高风险 RegistrationConversionMatrix / DepartmentConversionMatrix 的热力色（红→绿渐变）在 dark 背景下对比度不足，需单独定义 .dark 下的色阶变量。
涨跌颜色语义 中风险 代码中涨用 --chart-2（橙红）、跌用 --chart-3（绿），但 dark 下两者均已调整。需要 DeltaBadge 统一封装后统一验证。
Skeleton shimmer 中风险 Skeleton 的 shimmer 动画在 dark 下 from/to 颜色需从 gray-200/300 改为 neutral-700/600。
图表 tooltip 背景 中风险 Recharts 默认 tooltip 背景为白色，dark 模式应覆盖为 var(--popover)（#262626）。
阴影强度 低风险 dark 下 --elevation-sm 已从 0.1 调至 0.3，只要组件使用 shadow-[var(--elevation-sm)] 即可自动适配。
CompactHeader 下拉菜单 低风险 Select / DropdownMenu 的 Content 使用 --popover token，已自动适配，无需单独处理。
